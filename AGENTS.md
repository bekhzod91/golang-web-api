# AGENTS.md

This file is a roadmap to project documentation.

## Start Here

- `docs/ARCHITECTURE.md` — system design, layers, boundaries, request flow
- `docs/SETUP.md` — local/dev environment setup and run instructions
- `docs/API.md` — API conventions and OpenAPI workflow
- `docs/DB.md` — schema, migrations, and multi-tenant DB notes
- `docs/TESTING.md` — test strategy and how to run tests
- `docs/COMMIT.md` — commit/PR conventions

## Contribution Flow

1. Read `docs/ARCHITECTURE.md`
2. Read `docs/API.md` and `docs/DB.md` (if relevant)
3. Implement changes
4. Run tests (`docs/TESTING.md`)
5. Open PR following `docs/COMMIT.md`

## Note

If any referenced doc is missing, create it under `docs/` before making large changes.
