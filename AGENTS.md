Agent orientation and contribution roadmap for `golang-web-api`.

---

## Project at a Glance

Multi-tenant REST API in **Go 1.23** following **Clean Architecture** and **DDD**.
One PostgreSQL database, one schema per tenant, Redis for token storage, AWS S3 for file uploads.
HTTP via [chi](https://github.com/go-chi/chi), queries via [SQLC](https://sqlc.dev), contract-first API via OpenAPI 2.0.

## Layer Map — Where Does Code Go?

```
src/
├── domain/          ← business rules only (no DB, no HTTP, no frameworks)
│   ├── entity/      ← objects with identity: User, Role, Tenant
│   ├── value_object/← immutable typed values: Email, Password, Token …
│   ├── repository/  ← repository interfaces (no implementations here)
│   └── exception/   ← domain error types
├── application/
│   ├── command/     ← write use-cases: CreateUser, SignIn, DeleteRole …
│   └── query/       ← read use-cases:  GetMe, GetUsers, GetRoles …
├── infrastructure/
│   ├── api/handler/ ← thin HTTP handlers — parse DTO, call use-case, render
│   ├── api/dto/     ← AUTO-GENERATED by `make swag` — never hand-edit
│   ├── api/router/  ← route registration (public + tenant-scoped groups)
│   ├── storage/     ← repository implementations (DB + Redis)
│   ├── sqlc/        ← AUTO-GENERATED by `make sqlc` — never hand-edit
│   └── migrations/  ← shared/ and tenant/ SQL migration files
└── pkg/             ← framework-agnostic utilities (any layer may import)
```

**Quick placement checklist** (full rules in [`docs/DDD.md`](docs/DDD.md)):

- Rule about a single value → **Value Object**
- State change that must keep one entity consistent → **Entity method**
- Domain logic spanning multiple entities, no I/O → **Domain Service**
- Orchestrates steps + persistence for one user action → **Application use-case**
- Anything touching DB / Redis / HTTP / AWS → **Infrastructure**

## Contribution Flow

Follow these steps in order for any change:

1. **Orient** — read [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) to understand affected layers.
2. **Plan placement** — consult [`docs/DDD.md`](docs/DDD.md) before writing business logic.
3. **API changes** — edit specs in `src/openapi/`, then run `make swag` to regenerate DTOs.
   See [`docs/OPENAPI.md`](docs/OPENAPI.md).
4. **DB changes** — create a migration with `make migration-shared` or `make migration-tenant`,
   run `make sqlc` after editing `src/query.sql`.
   See [`docs/DATABASE_MIGRATION.md`](docs/DATABASE_MIGRATION.md).
5. **Implement** — domain → application → infrastructure (inward-first).
6. **Test** — run `make test`; add/update fixtures and test cases.
   See [`docs/TESTING.md`](docs/TESTING.md).
7. **Open PR** — follow branch and commit conventions in [`docs/COMMIT.md`](docs/COMMIT.md).

## Code Generation — Must-Know Rules

| Command | When to run |
|---|---|
| `make swag` | After any edit to `src/openapi/*.yaml` |
| `make sqlc` | After any edit to `src/query.sql` or migration files |
| `make migrate` | After adding new migration files |

> **Never hand-edit** files in `src/infrastructure/api/dto/` or `src/infrastructure/sqlc/`.
> They are overwritten on every `make swag` / `make sqlc` run.

## Multi-Tenancy Awareness

- Every tenant-scoped request requires the `X-Tenant: <schema_name>` header.
- The `MultiTenancy` middleware resolves the schema and injects a scoped `*sql.DB` — handlers and use-cases must not contain any tenancy logic themselves.
- Tests must always set `X-Tenant: test_tenant` (and `Authorization`) on protected endpoints.

## Key Interfaces (quick reference)

```go
// infrastructure/core/context.go
type IContext interface {
    Storage() storage.IStorage   // tenant-scoped repositories
    User()    *entity.User       // authenticated user
    Logger()  logger.ILogger
    AWS()     *aws.Client
    BindJSON(target any) error
    ShouldBindJSON(target interface{ Validate(...) }) error
    URLParam(key string) string
    QueryParam(key string) string
    TenantSchemaName() string
    JSON(status int, v any)
    OK(v any) | Created(v any) | NoContent()
    BadRequest(err) | NotFound() | Unauthorized() | Forbidden()
}

// infrastructure/storage/storage.go
type IStorage interface {
    Token() repository.ITokenRepository
    User()  repository.IUserRepository
    Role()  repository.IRoleRepository
}
```

## Guardrails

- **No business logic in handlers** — handlers parse the request and render the response only.
- **No DB/Redis/HTTP imports in `domain/`** — the domain layer must stay framework-free.
- **No stored procedures or triggers** — all business rules live in Go code.
- **Migrations must be backward compatible** — additive changes only; never drop or rename columns.
- **Tests are black-box** — test through HTTP endpoints, not by calling internal packages directly.
- **Do not create `docs/` files** unless explicitly asked — update existing ones instead.
